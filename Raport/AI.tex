We can mention here that we have used some AI, but at many points have we AVOIDED use of AI, in order to better learn the frameworks we are dealing with.
Learning React, REST, How to set up databases, Kotlin, etc. are not "part" of the course, but still something we have prioritized. This does, however, take time and leads to time-constraints.

After finishing coding the experience system there was need to check for errors.
At the start there was sent a few question to an AI (qwen-coder-2.7) in order for him to check naming, function structures and for logic.
He said everything looked good and that there were no errors.
However, when doing testing and manual testing, we found some errors.
First error came with testing, and it found that experience was not added correctly.
The manual testing was done to check the database and api.
Quickly figured out that the function which was adding xp to each student was not updating the database.
So, AI did not figure out two easy to check errors and was not part of generating the code and fixing errors before initial testing.
