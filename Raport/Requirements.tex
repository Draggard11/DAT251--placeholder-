\section{Requirements}

\subsection{Overview}
Based on the pre-study, a set of functional and non-functional requirements were derived
to address the challenges related to motivation, engagement, and learning.

\subsection{Functional Requirements}
A brief summary of functional requirements are listed here. Full user stories are included in the Appendix \ref{User Stories} %TODO check reference appendix or section 4
The requirements are derived from user needs identified in the personas.
For example, Fiona requires independent learning support, while Elias requires structure and guidance.
\begin{itemize}
    \item Students should be able to track their progress without relying solely on grades
    \item Students should be able to ask anonymous questions
    \item Students should be able to participate in study groups
    \item The system should provide feedback on participation
    \item The system should support gamification elements such as streaks and XP to increase engagement and motivation
\end{itemize}

\subsection{Non-functional Requirements}
\begin{itemize}
    \item Usability: The system must be intuitive and easy to use
    \begin{itemize}
        \footnotesize
        \item[] To compete with existing solutions to the problem, 
        we should avoid friction that could lead to users 
        prefering alternatives.
    \end{itemize}
    \item Accessibility: The system should work on common devices
    \begin{itemize}
        \footnotesize
        \item[] Similar to above. There also exists potential users without f.ex. smartphones or who would object to downloading our product that we want to accomodate.
    \end{itemize}
    \item Scalability: The system should support many users
    \begin{itemize}
        \footnotesize
        \item[] We believe some functionality of our product would work better with a larger user base, such as group-creators, competitions, and shared user-created quizzes.
    \end{itemize}
    \item Maintainability: The system should be modular and extensible
    \begin{itemize}
        \footnotesize
        \item[] Especially during prototyping/feedback cycles, it is important to easily accommodate for new feature creation/deletion/adjustments
    \end{itemize}
\end{itemize}

\subsection{Sustainability Considerations}
At this stage of the project, sustainability considerations are necessarily limited, as no concrete solution has yet been defined. 
Nevertheless, several relevant aspects have been identified.

From a technical perspective, the solution must align with users’ existing knowledge and access to technology, 
such as personal computers and mobile devices. 
This implies that any future system should be easy to use and accessible within realistic technical constraints.

From a social perspective, the project focuses on improving motivation, engagement, and encouragement within educational contexts. 
Addressing these issues has the potential to positively impact users’ well being and overall learning experience.

Regarding economic sustainability, no budget has been defined at this stage. 
The project will explore possibilities for making the solution freely available. 
No physical production is planned.

Environmental considerations include limiting or avoiding excessive use of resource intensive AI technologies, as high computational usage may have negative environmental impacts.
